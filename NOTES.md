# Accessible Component Gaps Analysis: Custom vs shadcn/ui 🔍

This document presents a comparative analysis between our custom-built ARIA components and the components generated by `shadcn/ui` (underpinned by Radix UI Primitives), identifying key architectural and accessibility gaps.

---

## 🚀 Identified Gaps

### 1. React Portals vs. In-Place DOM Rendering (z-index & Clip Clipping)
- **Our Custom Modal**: Renders inline directly within the parent component's DOM node hierarchy. This makes it susceptible to styling leaks, z-index layering conflicts, or layout clipping if a parent wrapper has `overflow: hidden`.
- **shadcn/ui Dialog**: Employs a Portal component (`<DialogPortal>`) which detaches the dialog overlay and content from the local render tree and appends it directly to the root of the document body (`document.body`). This guarantees that the dialog renders on top of all other elements, regardless of local DOM parent nesting.

### 2. Auto-Focus Guards & Strict Focus Trapping
- **Our Custom Modal**: Implements focus trapping using custom refs and a keydown listener listening to `Tab` key presses. While functional, it doesn't safeguard against auxiliary devices or focus redirection from external screen readers, and requires manual calculations to wrap active indices.
- **shadcn/ui Dialog**: Integrates Radix's FocusScope primitive which places invisible "focus guards" (elements with tabIndex 0) at the start and end of the portal DOM node. This ensures that even if keyboard focus is moved programmatically or via browser navigation, focus is immediately intercepted and routed back to the dialog, preventing focus escaping.

### 3. Scroll Locking & Layout Shift Prevention
- **Our Custom Modal**: Opening the modal does not prevent background scrolling. Users can still scroll the page behind the backdrop using the mouse wheel.
- **shadcn/ui Dialog**: Automatically locks body scroll by injecting a `style="pointer-events: none; overflow: hidden;"` attribute on the root `<html>` element upon mount. Additionally, it calculates and injects style offsets (padding compensation) to prevent the website layout from shifting when the scrollbar disappears.

### 4. Semantic Linking via Context ID Generation
- **Our Custom Modal**: Uses static hardcoded string IDs (e.g. `id="custom-modal-title"`) to bind `aria-labelledby` and `aria-describedby` elements. If multiple modals are mounted, this results in duplicate ID violations.
- **shadcn/ui Dialog**: Utilizes dynamic React Context IDs. The `DialogTrigger`, `DialogTitle`, and `DialogDescription` dynamically sync and generate randomized unique IDs under the hood to ensure compliance with the HTML/W3C unique ID spec, even in multi-instance applications.
